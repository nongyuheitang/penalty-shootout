# 世界杯点球大战 - 开发提示词记录

## 项目概述
HTML5 Canvas 双人对战点球游戏，支持键盘/鼠标、Joy-Con (WebHID)、Gamepad API 三种输入方式，10 轮交替攻防。

**线上地址**：https://nongyuheitang.github.io/penalty-shootout/
**本地运行**：双击 `启动.bat`，浏览器访问 http://localhost:3000

---

## 初始需求

```
帮我做一个游戏：世界杯点球大战 HTML5 Canvas 网页游戏
- 双人模式：P1（踢球方）瞄准+蓄力+射门，P2（守门方）左右移动扑救
- 三种输入：鼠标/触控、键盘（P1: WASD+Space, P2: 方向键）、Joy-Con（WebHID）
- 完整动画链：idle→aim→charge→runUp→kick→ballFly→netHit→result→reset
- 视觉效果：球场氛围、贝塞尔球轨迹、进球粒子、屏幕震动、防守范围可视化
- 5 球/局，最终比分
- Vite + 原生 JavaScript，无外部图片
```

### 3. 键盘输入修复
```
怎么键盘操作不了了
键鼠还是没反应
```
- **根因**：InputManager 类与全局键盘事件冲突
- **修复**：完全重写输入架构，键盘直接用 `window.addEventListener` 存入 `KEY` 对象，main.js 每帧构建 `inp` 纯对象，不再经过任何输入类

### 4. FIFA 风格瞄准
```
学习fifa那样只显示大致的朝向，显示蓄力条控制力度，再左右摇摆
守门员用摇杆在整个守门区域用点来防守，灵敏度调高
```
- 去掉贝塞尔预测线，改为**方向锥形**（半透明锥形区域 + 中心虚线 + 箭头）
- 蓄力条自动摆动（0→100→0），松手锁定力度
- 守门员改为**二维自由移动**（不再限于左右），方向键 ↑↓←→ 在球门区域内移动
- 判定改为欧几里得距离
- 守门员速度 500 px/s

### 7. Joy-Con 摇杆问题修复
```
右边手柄的摇杆完全没反应
摇杆漂移
摇杆操控的时候会漂移
```
- **右摇杆无反应**：代码只取了 `analogStickLeft`，右手柄的摇杆在 `analogStickRight`。改为根据手柄类型（`isLeft`）优先取正确属性
- **漂移**：死区 0.30 + **n² 二次曲线**强力压制死区边缘的微小漂移
- **自动校准**：连接后采集前 60 个样本计算中心偏移，后续读数减去偏移再进死区
- 校准完成后控制台打印：`P2 校准完成! 中心偏移: (0.032, -0.351)`

### 8. 手柄/鼠标互不干扰
```
手柄操作的时候会和鼠标的输入互相干扰，在摇杆移动的时候禁用鼠标输入
连接手柄的那一刻就禁用键鼠
```
- 检测到 Joy-Con 连接 → **完全禁用**对应玩家的键鼠输入
- `shooterHasJC` / `keeperHasJC` 布尔值控制是否跳过键鼠分支

## 常见问题与修复

### 键盘/鼠标没反应
1. 检查是否连了 Joy-Con → 手柄连接后自动禁用对应玩家的键鼠
2. 检查右下角/左下角操作提示的颜色与内容

### 摇杆漂移
1. 连接后**手不碰摇杆等 1 秒**，让校准完成
2. 控制台查看校准偏移量（`P2 校准完成! 中心偏移: (x, y)`）
3. 若漂移仍存在，在 constants.js 提高 `JOYSTICK_DEADZONE`

### 右手柄无反应
1. 检查 WebHID 弹窗是否选中了两个手柄
2. 控制台查看是否有 `右手柄 → P2` 的初始化日志
3. 查看事件 #1 的完整数据 dump，确认摇杆属性名

### 只有一个手柄工作
1. WebHID 每次弹窗只能选**一个**手柄，需要**点两次**"连接 Joy-Con"按钮
2. 先连 P1，按钮变为"连 P2"，再点一次连 P2

### 攻防切换后操作混乱
1. 检查顶部记分牌的 ⚽ 指示器在正确的一方
2. 检查左下角/右下角操作提示的键位说明
